# 外部播放器注入

外部播放器注入系统允许用户在 Emby Web UI 中一键调用本地播放器（如 PotPlayer、VLC、IINA 等）播放 STRM 媒体文件。

## 为什么需要外部播放器？

Emby Web UI 的浏览器播放器受限于浏览器解码能力和 CORS 策略，对某些格式支持不理想。外部播放器（特别是 PotPlayer、VLC）解码能力更强，且没有跨域限制，可以直接处理 302 重定向。

## 注入机制

### 1. JS 脚本注入

反向代理拦截 Emby Web UI 的 HTML 响应，在 `</head>` 前注入一段 JavaScript。该脚本在媒体详情页面为每个已配置的外部播放器添加一个按钮。

渲染后的按钮样式：

```
[PotPlayer][VLC][IINA][...]
```

用户点击按钮时，脚本通过 `window.open()` 或修改 `location.href` 启动自定义协议 URL。

### 2. API 响应注入

反向代理拦截 `/Users/{user_id}/Items/{item_id}` 的 API 响应，在 `MediaSources[].MediaAttachments` 数组中注入 `ExternalUrls` 条目。

每个播放器的 ExternalUrl 格式：

```json
{
  "Name": "PotPlayer",
  "Url": "potplayer://http://proxy:8097/videos/...?MediaSourceId=...&api_key=...",
  "Description": "在 PotPlayer 中播放"
}
```

### 3. 重定向端点

代理提供一个 `/redirect2external?link=<base64>` 端点，解码 base64 后的 URL 并返回 302 重定向。JS 脚本通过这个端点间接跳转，避免直接暴露策略 URL。

## 播放协议

### PotPlayer（Windows）

```
potplayer://<stream_url>
```

### VLC（多平台）

```
# Windows
vlc://<stream_url>

# macOS
vlc://<stream_url>

# iOS
vlc-x-callback://x-callback-url/stream?url=<encoded_url>

# Android
intent://<stream_url>#Intent;package=org.videolan.vlc;end
```

### IINA（macOS）

```
iina://weblink?url=<encoded_url>
```

### Infuse（iOS/tvOS）

```
infuse://x-callback-url/play?url=<encoded_url>
```

### MPV

```
mpv-handler://play/<base64_url>
```

### Android 通用

```
intent://<stream_url>#Intent;package=<pkg_name>;end
```

## 配置

在管理 Web UI 的 Emby 配置中启用"外部播放器"功能，选择需要注入的播放器列表。如果列表为空，默认注入所有支持的播放器。
